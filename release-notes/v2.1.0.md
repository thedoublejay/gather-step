# Gather Step v2.1.0

Onboarding, generated AI context, Python parsing parity, and projection-impact release.

`v2.1.0` polishes the v2 onboarding path and generated AI context, promotes Python to first-class parsing alongside TypeScript and JavaScript, ships static projection-impact tracing, and adds an opt-in TUI dashboard.

## Highlights

- Made `gather-step init` the primary setup path in docs and landing copy, with a workspace directory diagram and explicit prompt defaults.
- Updated init output casing to "Gather Step" and made the local MCP default visible in the interactive prompt.
- Promoted Python to first-class parsing alongside TypeScript and JavaScript.
- Added projection-impact tracing for derived fields, persisted projections, filters, indexes, and backfills.
- Added an opt-in `gather-step tui` dashboard subcommand backed by a 738-line ratatui frontend.
- Generated rule and Claude/AGENTS workspace context now ship as opt-in artifacts during `gather-step generate`.
- Added a CODEOWNERS generator surfaced through `gather-step generate codeowners` for repos with ownership data.
- Refreshed CLI output modes with a Braille spinner, Unicode block-fill progress bar, and aligned ASCII banner padding.
- Bumped the app, Cargo workspace, internal crate dependency versions, and website package metadata to `2.1.0`.

## Core Product Changes

### Projection Impact

- Added the `projection-impact --target <FIELD>` CLI command and `projection_impact` MCP tool for static field-level projection tracing, including `evidence_verbosity` controls for summary versus full evidence.
- Added `DataField` graph nodes plus `ReadsField`, `WritesField`, `DerivesFieldFrom`, `FiltersOnField`, `IndexesField`, and `BackfillsField` edges.
- Planning and change-impact packs can now include short projection hints and `projection_impact:*` gap markers while the full evidence stays behind the dedicated projection tool.
- Added oracle, CLI/MCP serialization, integration, and parser extraction-fidelity coverage for projection chains, Mongo-style mappings, JSON/YAML index mappings, and false-positive fixtures.
- v2.1 projection impact changes the generated graph schema. Existing `.gather-step` storage should be rebuilt with `gather-step reindex` before relying on projection-impact output.
- Projection impact intentionally does not infer deployed runtime ownership. Verify deployment owners separately when duplicate or transitioning services exist; the planned runtime-ownership guardrail (Workstream B in the projection-impact plan) is deferred to a future release.

### Python parsing parity

- Promoted Python to first-class parsing alongside TypeScript and JavaScript.
- Added Python sibling-import resolution, qualified nested function names, configured src-package layout support, and Python class-relationship preservation.
- Added Python planning oracle checks and bench storage measurement.
- Added Python import-resolution regression suite covering sibling imports, src-layout packages, imported module calls, and qualified import heads.

### Onboarding and output

- Made `gather-step init` the primary setup path; added the workspace directory diagram, explicit prompt defaults, and the visible local MCP default.
- Added an opt-in TUI dashboard via `gather-step tui` for an interactive multi-pane workspace view of repos, symbols, routes, and events. The default `gather-step` invocation continues to print the no-args summary; the TUI is only entered when explicitly requested.
- Polished CLI output modes with a Braille progress spinner, Unicode block-fill progress bars, and ASCII banner padding.
- Added `feat(output): generate Claude rules and manifesto context` — `gather-step generate` now produces Claude-targeted workspace summaries and rule files.
- Generalized privacy cleanup references so generated docs and onboarding copy stay free of repo-specific or private-process wording.

### Website and documentation

- Added the language support page documenting first-class parsing for TypeScript, JavaScript, and Python.
- Marked v2.0.0 as released and logged v2.1.0 Python work in the changelog.
- Refreshed the projection benchmark documentation with diagnostics, fresh-release results, and benchmark-head pointers.
- Documented TUI setup and added CI routing for the TUI workflow.
- Bumped the website Astro toolchain.

## Upgrade Notes

- Existing workspaces can keep their `gather-step.config.yaml`.
- v2.1 projection impact changes the generated graph schema. Rebuild `.gather-step` state before relying on projection-impact output:

```bash
gather-step --workspace /path/to/workspace reindex
```

- If a watch process is running, stop it before the rebuild and restart it afterward.
