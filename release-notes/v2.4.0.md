# Gather Step v2.4.0

Release date: 2026-05-01
Status: released

`v2.4.0` focuses on making setup and indexing feel predictable for real multi-repo workspaces. Existing configs are now respected, repo selection happens once at the start of onboarding, progress copy is clearer, and final indexing output includes duration and index size.

## Highlights

- `gather-step init` now reuses an existing `gather-step.config.yaml` instead of blocking the flow or reintroducing repos you already removed.
- Interactive setup starts with a numbered checkbox-style repo picker, including `all` and `none` shortcuts.
- Existing repo config metadata is preserved for selected repos, including custom names, depth settings, provider settings, and indexing rules.
- `gather-step watch N` stops after `N` completed indexing runs.
- Indexing progress now uses a shorter bar, shows the repo path being indexed, and prints elapsed time plus index size at the end.
- Finalization copy now uses polished sentence case: `Flushing search index...`, `Counting cross-repo edges...`, and `Precomputing N context packs...`.
- Indexing emits clear start and finish logs for workspace and per-repo indexing.
- Static JSON/YAML mapping files no longer go through SWC, removing noisy recovered-parse warnings.
- Ambiguous sibling Python packages are left unresolved instead of choosing a first match and warning noisily.
- AI Markdown generation now shows a loader while files are being generated.
- Setup completion now includes a one-line next step and the MCP tools docs link: <https://gatherstep.dev/reference/mcp-tools/>.
- The docs CLI reference layout has been constrained so body content does not overlap under the sidebar.

## Upgrade Notes

- Existing workspaces can keep their `gather-step.config.yaml`.
- If you previously removed repos from the config, run `gather-step init` and press Enter at the repo-selection step to keep that selection.
- Rebuild generated state after upgrading:

```bash
gather-step --workspace /path/to/workspace reindex
gather-step --workspace /path/to/workspace generate claude-md
gather-step --workspace /path/to/workspace generate agents-md
```
