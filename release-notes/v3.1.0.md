# Gather Step v3.1.0

Indexing-performance, storage, and security release. Replaces the SWC-based
TypeScript and JavaScript visitor with an Oxc-driven implementation, lands a
series of throughput and footprint compactions, and hardens the watch and
config surface against symlink and oversize-file abuse. The Phase 0 cold
RegASK baseline is captured in [`v3.1-baseline.md`](./v3.1-baseline.md).

## Highlights

- TypeScript and JavaScript parsing now runs on Oxc. SWC, its dependency tree,
  and the 3.4k-line visitor that drove it have been removed. Production
  symbol, edge, decorator, call-site, and constant-string outputs match the
  prior release modulo a function-signature accuracy fix.
- Indexing pipeline avoids redundant cloning, caches path-alias discovery,
  gates framework augmenters by language, skips the default index-size walk,
  bounds analytics queue depth and pack-target precompute by repo count, and
  moves git analytics off the writer hot path.
- Search index footprint drops several megabytes by replacing stored fields
  with fast-field decoding, compacting edge metadata tags, swapping the
  redundant per-row `description` text for a dedicated indexed-only
  `qualified_name` field, replacing the edge-kind index with counters, and
  storing 16-byte BLAKE3 prefixes for the per-file change-detection cache.
- Symlink-aware path guards reject symlinked deployment configs, ignore
  symlinked watch event paths, separate `git worktree add` arguments, and cap
  local config reads to bound parser memory on hostile inputs.
- Search query parser splits identifier separators (`-`, `_`, `.`, `/`) before
  parsing so snake-case and slash-bearing repo names tokenize the same way
  they index. Qualified impact queries fall back to the tail segment when the
  qualified form does not hit the search index.
- Workspace registry counts are refreshed from the final graph at the end of
  an index run so the registry never drifts behind the graph after partial
  failures or self-healing reindexes.
- Cargo workspace, internal crate dependency versions, and website package
  metadata bumped to `3.1.0`. Phase 0 dependency pins are unchanged: `redb`
  `=4.1.0`, `tantivy` `=0.26.1`, `dhat` `=0.3.3`. SWC pins (`swc_common`,
  `swc_ecma_ast`, `swc_ecma_parser`) are removed; Oxc pins (`oxc_allocator`,
  `oxc_ast`, `oxc_parser`, `oxc_span`) stay at `=0.128.0`.

## TypeScript and JavaScript parser

- Replaced the SWC visitor with an Oxc-driven implementation. The new visitor
  produces the same `ParseState` writes (NodeIds, edges, decorators, call
  sites, constant strings) as the previous backend so downstream consumers
  (storage, framework augmenters, snapshot tests) see no behavioural change
  beyond the function-signature accuracy fix.
- Removed `swc_common`, `swc_ecma_ast`, and `swc_ecma_parser` from the
  dependency tree, plus the entire `ts_js_swc` module. The Oxc adapter is
  the only production backend; `TsJsParserBackend::current()` returns
  `Oxc` and the dispatch table only has an Oxc arm.
- Added an `oxc_test_support` surface that mirrors the helpers test suites
  previously imported from `swc_test_support` (`parse_recovery_status_for_*`,
  `parse_full_pipeline_contains_symbol`, `parse_source_contains_ident`,
  `parse_ts_file_via_extension`, `top_level_declared_names_for_path`,
  `parse_import_bindings_for_path`).
- Function signatures emitted for zero-parameter methods are now precise
  (`handle()` instead of accidentally swallowing a preceding decorator
  argument such as `('build')` because of SWC's source-text fallback). Five
  extraction-fidelity snapshots updated to reflect the corrected output.

## Indexing performance

- Bounded context-pack precompute and pack-target selection by repo count.
- Cached path-alias discovery for the duration of an index run.
- Gated framework augmenters by language so non-TS/JS repos do not pay for
  TS-only augmentation traversals.
- Skipped the size-only filesystem walk on the default index path; size
  sampling is now opt-in.
- Avoided cloning traversal source bytes when the indexer can borrow the
  original buffer.
- Moved git analytics off the writer hot path and bounded its queue depth by
  repo count so storage commits no longer block on history work.

## Storage compactions

- Dropped the redundant search `description` text field; reintroduced
  `qualified_name` as a dedicated indexed-only field with a lighter
  tokenizer chain (SimpleTokenizer + LowerCaser, no stemming, no positions).
  Restores lowercase identifier matches (`createorder` -> `createOrder`) and
  repo-prefix queries (`service-a`) without bringing back the
  natural-language tokenizer chain. `SEARCH_INDEX_VERSION` is bumped to `1`.
- Decoded `is_exported` and `lang` from search fast fields instead of stored
  fields.
- Replaced the `edges_by_kind` projection with counters and compacted edge
  metadata tags.
- Truncated `file_index_state.content_hash` to a 128-bit BLAKE3 prefix for
  the per-file change-detection cache. Incremental classification truncates
  snapshot hashes through `metadata::stored_content_hash` before comparing
  so the prefix store stays consistent across read paths.
- Pruned stale context packs on write and salted cache keys by compatibility
  so format-version changes invalidate without manual cleanup.
- Generated schema baselines are reset for v3.1's fresh-release stance:
  `METADATA_SCHEMA_VERSION = 0`, `SEARCH_INDEX_VERSION = 1`, graph schema
  version `0`. None of these carry migration or upgrade branches.

## Security

- Watcher ignores symlinked event paths so symlink-cycles cannot drive
  unbounded reindex loops.
- `gather-step.local.yaml` and other local config reads are capped at a
  bounded byte budget so hostile configs cannot blow up parser memory.
- `git worktree add` arguments are passed positionally rather than glued
  into one shell string, removing a path-injection surface.
- Deployment topology config rejects symlinked paths so a hostile workspace
  cannot point the topology at out-of-tree YAML.

## Bug fixes

- Search queries split identifier separators (`-`, `_`, `.`, `/`) before
  parsing so doctor and search projection queries that previously missed
  snake-case symbols hit them again.
- Qualified impact queries fall back to the tail segment after the search
  compaction, restoring `Module::Symbol`-style lookups.
- Workspace registry counts are refreshed from the final graph at the end of
  an index run.
- Incremental classification truncates new content hashes to the stored
  prefix length before comparing.

## Test coverage

- Oxc parser self-validation guards across every TS/JS extraction fixture:
  parser status, top-level declared names, and import-binding shape.
- Secret-surface MCP smoke test exercises the redaction surface end-to-end.
- Deployment-topology MCP tools test pins the public response shape.
- Benchmark harness samples resource peaks (max RSS, peak memory footprint,
  open FDs on Unix) so future v3.1 perf claims can be validated
  quantitatively.

## Upgrade notes

- Existing `.gather-step` storage is invalidated by the search index version
  bump. Run `gather-step reindex` to rebuild the search store; metadata and
  graph stores reuse their on-disk format.
- Workspaces that pinned `gather-step-*` crates as path dependencies should
  bump their version constraints from `=3.0.0` to `=3.1.0`.
- The `swc_*` crates are no longer pulled in. Tooling that depended on the
  old `swc_test_support` re-export should switch to `oxc_test_support`,
  which exposes the same shapes but operates on Oxc spans.
